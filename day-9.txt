KEYS IN SQL
-------------------------------------------------
A Key is a single column or a combination of multiple columns in a table used to uniquely identify records.
Keys are also used to create relationships between different database tables.

Types of Keys:
1. Primary Key
2. Foreign Key
3. Composite Key
4. Unique Key
5. Alternate Key


1. PRIMARY KEY
-------------------------------------------------
A Primary Key is a column or set of columns that uniquely identifies each record in a table.

Properties:
• Each table can have only one Primary Key
• It must contain unique values
• It cannot contain NULL values
• It uniquely identifies every row in the table

Syntax:
CREATE TABLE department(
    id INT(10),
    name VARCHAR(50),
    sal VARCHAR(50),
    PRIMARY KEY(id)
);

Definition:
Primary Key is a column or set of columns that uniquely identifies each record in a table and does not allow duplicate or NULL values.


2. FOREIGN KEY
-------------------------------------------------
A Foreign Key is used to create a relationship between two tables.
It is a column in one table that refers to the Primary Key of another table.

Properties:
• Creates a relationship between tables
• Maintains data integrity
• Prevents invalid data insertion
• A table can have multiple foreign keys
• It can contain duplicate values

Syntax:
CREATE TABLE domain(
    id INT(10),
    name VARCHAR(50),
    contact VARCHAR(20),
    PRIMARY KEY(id),
    FOREIGN KEY(name) REFERENCES department(id)
);

Definition:
A Foreign Key is a column in a table that refers to the Primary Key of another table to establish a relationship between the two tables.


3. COMPOSITE KEY
-------------------------------------------------
A Composite Key is a Primary Key formed by the combination of two or more columns.
It is used when a single column cannot uniquely identify records.

Properties:
• Uses multiple columns
• Ensures unique identification
• Also called Compound Key

Syntax:
CREATE TABLE demo(
    id INT(10),
    name VARCHAR(50),
    contact VARCHAR(20),
    PRIMARY KEY(id, name)
);

Definition:
Composite Key is a primary key formed by combining two or more columns to uniquely identify a record.


4. UNIQUE KEY
-------------------------------------------------
A Unique Key ensures that all values in a column are unique.

Properties:
• Prevents duplicate values
• Allows only one NULL value
• A table can have multiple UNIQUE keys
• Similar to Primary Key but allows NULL

Syntax:
CREATE TABLE batch(
    id INT(10),
    name VARCHAR(50),
    contact VARCHAR(20),
    UNIQUE(id)
);

Definition:
Unique Key is a constraint used to ensure that all values in a column are unique and prevents duplicate entries in a table.


5. ALTERNATE KEY
-------------------------------------------------
An Alternate Key is a candidate key that is not selected as the Primary Key.

When a table has multiple candidate keys, one is chosen as the Primary Key and the remaining candidate keys become Alternate Keys.

Properties:
• Not chosen as Primary Key
• Still uniquely identifies records
• Implemented using UNIQUE constraint
• A table can have multiple Alternate Keys

Definition:
Alternate Key is a candidate key that is not selected as the Primary Key but can still uniquely identify records in a table.
